# Patch Note Guidelines

1. **Be concise:** Use minimal words.
2. **Start with action verbs:** Fixed, Added, Improved, Removed.
3. **Focus on impact:** State exactly what changed for the user.
4. **No fluff:** Remove filler, narrative, and redundancy.
5. **Group changes:** Use the categories below.

## Template

## [Version] - YYYY-MM-DD

### Features
- [description of new functionality]

### Bug Fixes
- [description of resolved issue]

### Improvements
- [description of enhancement]

### Internal Changes
- [description of refactoring or chore]
